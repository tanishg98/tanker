Agents

Agents run autonomously on a focused task and return a structured report. All agents are read-only — they never modify files.

🔍
explore read-only

Codebase exploration. Given a feature or bug, reads relevant files, traces data flow, identifies integration points and constraints, returns an exploration report with numbered open questions. Feeds directly into /createplan.

Read Grep Glob
✅
review read-only

Focused code review. Checks logging, error handling, TypeScript, security, React hooks, performance, and architecture. Every finding gets a severity (CRITICAL–LOW) and every CRITICAL/HIGH gets a concrete fix. Classified as AUTO-FIX or ASK.

Read Grep Glob
🎨
site-eval read-only

Pre-launch evaluator for static websites. Audits 9 dimensions — completeness, typography, responsiveness, images, performance, and AI slop detection. Issues PASS / CONDITIONAL / FAIL verdict. No site ships without a PASS.

Read Grep Glob

pre-merge new

Mandatory combined quality gate. Runs both code quality review and deep bug analysis (autoresearch-review) in a single pass. Returns BLOCK / PASS WITH FIXES / PASS. Run before every PR merge — no exceptions.

Rules

Always-on. Load automatically in every session. No invocation needed.

⚡ builder-ethos always on
01
Boil the Lake
AI makes completeness near-zero-cost. Do the full implementation every time — 100% of spec, all edge cases, accessible, responsive from commit one.
02
Search Before Building
Three layers: tried-and-true patterns → new-and-popular (scrutinise) → first principles. The goal is the Eureka Moment: finding where conventional assumptions break for your case.
03
Fix-First Review
Every finding is AUTO-FIX (apply immediately, no discussion) or ASK (propose, confirm first) before any action is taken.
04
No AI Slop
Explicit ban list: purple gradients, 3-column feature grids, centred text >60%, bubbly border-radius, generic hero copy, blacklisted fonts, missing focus states.
⌨️ code-standards always on
Types
No untyped code
No any without a comment. Proper interfaces for anything shared between files.
Comments
Why, not what
Comments explain the reasoning. The code explains what it does. // increment counter is noise.
Patterns
Follow the codebase
Find the closest existing example and follow it exactly — naming, file structure, export patterns, error handling style.
🌐 static-site-standards always on · static sites only
Architecture
Single-file first
One index.html with inline CSS/JS. No frameworks, no build tools. CDN for external deps only.
CSS
Custom properties always
Colors, fonts, spacing defined in :root. Mobile-first. clamp() for fluid type. No inline styles.
Eval Gate
site-eval before shipping
No site is done until the site-eval agent returns PASS. CONDITIONAL or FAIL means more work first.
